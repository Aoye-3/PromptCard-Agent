## 2. 背景与问题

### 2.1 背景

Seedream 一期已经具备图片生成领域契约、模型连接、OS Keyring、Ark SDK adapter、生成编排、永久历史、媒体资产闭环以及部分前端组件。2026-07-15 的真实桌面端前端检查显示：代码层存在的能力尚未形成用户可发现、可配置、可完成的产品路径。

### 2.2 当前用户问题

1. 自由画布工具栏没有可见的“图片生成”入口，用户无法创建真实图片生成节点。
2. 图片节点入口受 Feature Flag 控制且被完全隐藏，用户无法判断功能未开启、未配置还是不存在。
3. Agent 面板中的“模型服务”将文字模型、图片模型、连接和默认槽位放在同一界面，用户难以建立任务心智。
4. 当前页面暴露 `chat.primary`、`image.primary` 等内部术语，没有以“默认文字模型”“默认图片模型”表达用户目标。
5. 火山方舟 Provider、Ark SDK、API Key、连接、Seedream 模型和默认分配之间的关系没有图形化呈现。
6. 即使后端能力存在，用户仍无法顺畅完成“配置图片模型 → 创建节点 → 连接输入 → 生成 → 复用结果”的闭环。

### 2.3 产品判断

本 PRD 采用以下完成标准：

- 组件存在但没有用户入口，不算完成。
- 入口存在但无法完成配置或生成，不算完成。
- 仅在测试或开发者环境可调用、用户界面无法理解状态，不算完成。
- 只有真实用户可以从界面完成端到端任务，才算前端接入完成。

## 3. 产品目标与成功标准

### 3.1 产品目标

1. 用户可以从自由画布工具栏明确创建图片生成节点。
2. 用户可以分别进入“文字模型”和“图片生成模型”管理界面。
3. 用户可以通过图形化界面管理火山方舟连接、凭据状态、Ark SDK 状态、Seedream 模型能力和默认图片模型。
4. 用户可以在画布中完成提示词、快捷提示词、主图、多参考图、结构化 `@` 引用、智能改图和点/框区域修改。
5. 用户可以理解生成状态、错误、历史、媒体资产和再次生成之间的关系。
6. 前端不暴露密钥、供应商临时 URL、内部存储路径或不必要的后端术语。

### 3.2 产品验收指标

- 新用户无需阅读技术文档即可从自由画布找到“图片生成”入口。
- 未配置图片模型时，点击入口可以进入明确的配置引导，不出现静默隐藏或无反馈。
- 已配置默认图片模型时，用户可以在一个连续流程内创建节点并发起生成。
- 文字模型和图片生成模型的连接、默认分配和状态不会互相混淆。
- 生成成功后，结果同时可从节点、历史和媒体库访问。
- 生成失败时，界面提供可执行的恢复动作，不只显示后端错误码。
- 所有 P0、P1 验收场景通过自动化测试和桌面端人工验收。

### 3.3 守护指标

- 浏览器、项目 JSON、SQLite、日志和错误提示中不得出现真实 API Key。
- 不允许前端直接调用 Ark SDK 或供应商图片 API。
- 不允许为了展示进度而伪造百分比、取消能力或多图输出。
- 不允许未配置模型时直接隐藏核心入口。

## 4. 范围

### 4.1 本期范围

- 自由画布“图片生成”工具栏入口与节点创建。
- 图片节点的端口、连线、Inspector、提示词、参考图、尺寸、模式和区域编辑交互。
- Agent 面板信息架构拆分为“文字模型”和“图片生成模型”两个入口。
- 火山方舟/Seedream 图形化 Provider、SDK、连接、凭据、模型能力和默认分配管理。
- 生成状态、错误恢复、结果操作、永久历史和媒体复用交互。
- Feature Flag 的可见状态、启用策略和未启用提示。
- 相关空状态、加载状态、错误状态、禁用状态和可访问性。

### 4.2 非本期范围

- 第二个真实图片供应商。
- 原生二进制 mask 上传。
- 4K、流式百分比、多图批量输出和供应商任务取消。
- 任意 Python 包名或任意版本的通用包管理器。
- 云端团队同步、计费充值和供应商账号管理。
- 移动端完整画布编辑体验。

### 4.3 假设与约束

- 第一期图片模型为 `doubao-seedream-5-0-pro-260628`。
- 每次请求生成一张图片。
- 图片输入总数不超过十张。
- 区域修改使用 point/bbox 指代，不宣称为原生 mask。
- API Key 由后端写入 OS Keyring；前端只显示 `credentialConfigured`。
- 前端入口受产品 Feature Flag 控制，但关闭时必须提供可理解状态，不能静默消失。
- SDK 版本由受信任的后端依赖锁定；前端展示真实状态并可触发受控检测。是否提供受控一键修复见“待确认问题”。

## 5. 目标用户与核心场景

### 5.1 目标用户

- 使用提示词和参考图进行视觉创作的创作者。
- 需要在画布中反复生成、改图和组合素材的内容生产者。
- 负责配置本地模型连接的应用管理员或高级用户。

### 5.2 核心用户故事

- 作为创作者，我希望在画布工具栏直接创建图片生成节点，而不是寻找隐藏设置。
- 作为创作者，我希望将快捷提示词、文本节点和参考图连接到生成节点，并理解每条连接的作用。
- 作为创作者，我希望通过 `@` 指定不同参考图承担不同角色。
- 作为创作者，我希望在同一个节点内切换文生图、参考图生成、智能改图和局部修改。
- 作为管理员，我希望分别管理文字模型和图片模型，不混淆对话与图片生成用途。
- 作为管理员，我希望看到 Ark SDK、火山方舟连接、凭据和 Seedream 模型是否可用。
- 作为创作者，我希望失败后知道应该补输入、配置凭据、等待容量还是重试。
- 作为创作者，我希望历史结果永久保留，并可重新放入画布或继续改图。

## 6. 信息架构

### 6.1 Agent 面板导航

当前“模型服务”入口调整为两个一级入口：

1. “文字模型”
2. “图片生成模型”

“默认模型”不再作为与连接管理脱离的技术页面；默认分配分别放在对应模型页面中：

- “文字模型”页面管理“默认文字模型”。
- “图片生成模型”页面管理“默认图片模型”。

内部标识 `chat.primary`、`image.primary` 仅可在开发者信息、日志诊断或辅助说明中出现，不作为主要标题。

### 6.2 自由画布工具栏

底部工具栏至少包含：

- “文字”
- “卡片/提示词”
- “图片”
- “图片生成”

“图片”表示上传或放置现有图片；“图片生成”表示创建 AI 图片生成节点。两者必须使用不同图标、Tooltip 和可访问名称。

### 6.3 图片生成节点信息结构

节点卡片展示高频信息：

- 模式
- 模型
- 主要提示词摘要
- 输入图片数量
- 尺寸/分辨率
- 当前状态
- 当前结果缩略图
- “生成/再次生成”主操作

低频设置进入右侧 Inspector：

- 连接与模型切换
- 完整提示词编辑
- 参考图排序
- 输出格式和水印
- 区域编辑
- 历史入口

## 7. 端到端目标流程

1. 用户进入 Agent 面板的“图片生成模型”。
2. 用户选择“火山方舟”。
3. 界面展示 Ark SDK 状态、版本和兼容性。
4. 用户创建连接，输入连接名称和 API Key。
5. 后端安全保存凭据，前端仅显示“凭据已配置”。
6. 用户执行“测试连接”。
7. 用户查看 Seedream 5.0 Pro 能力并设为“默认图片模型”。
8. 用户进入自由画布，点击“图片生成”。
9. 系统在视口中心创建并选中节点，同时打开 Inspector。
10. 用户输入或连接提示词，上传或连接参考图，并使用 `@` 选择图片角色。
11. 用户选择模式、尺寸、格式、水印和可选区域。
12. 用户点击“生成”。
13. 节点展示真实状态；成功后显示本地结果。
14. 结果自动进入永久历史和媒体库。
15. 用户可再次生成、智能改图、作为新图片节点或作为其他生成节点参考图。

## 8. 功能需求

### 8.1 P0：画布图片生成入口

#### 8.1.1 入口展示

- 在自由画布底部工具栏增加“图片生成”按钮。
- 按钮始终具有可访问名称和 Tooltip：“创建图片生成节点”。
- 前端 Feature Flag 开启时按钮可用。
- 产品未开放时按钮可隐藏；开发/测试环境默认应可开启。
- 功能已开放但未配置图片模型时，按钮保持可见。

#### 8.1.2 创建行为

- 点击按钮后，在当前视口中心创建节点。
- 支持从工具栏拖拽到画布指定位置创建节点。
- 创建成功后自动选中新节点并打开 Inspector。
- 已存在默认图片模型时，自动绑定默认连接和模型。
- 未配置默认图片模型时，创建引导态节点，不写入无效 `connectionId`。

#### 8.1.3 未配置引导

- 节点展示标题：“尚未配置图片生成模型”。
- 主按钮：“前往配置”。
- 点击后进入 Agent 面板“图片生成模型”，并保留返回当前项目的上下文。
- 配置完成返回画布后，节点重新读取默认图片模型。

#### 8.1.4 空画布引导

- 当项目没有任何节点时，可在画布空状态展示“创建图片生成节点”快捷操作。
- 空状态不能遮挡基础画布操作和底部工具栏。

### 8.2 P0：文字模型管理页面

#### 8.2.1 页面入口

- Agent 面板二级导航增加“文字模型”。
- 页面副标题说明：“管理聊天 Agent 和文本任务使用的模型连接”。

#### 8.2.2 页面结构

- 顶部：状态摘要与“新建文字模型连接”。
- 主区域：文字模型连接列表。
- 详情区：选中连接的配置、凭据状态和测试结果。
- 页面底部或固定区域：“默认文字模型”。

#### 8.2.3 连接卡片

- 展示连接名称、Provider、启用状态、凭据状态、最近测试状态和默认标记。
- 行内操作：“编辑”“测试”“设为默认”“删除”。
- DeepSeek 连接只展示文字模型能力，不出现图片能力。

#### 8.2.4 默认文字模型

- 用户文案使用“默认文字模型”。
- 选择项显示“连接名称 / 模型名称”。
- 未分配时显示风险提示：“Agent 对话暂不可调用模型”。

### 8.3 P0：图片生成模型管理页面

#### 8.3.1 页面入口

- Agent 面板二级导航增加“图片生成模型”。
- 页面副标题说明：“管理图片 Provider、SDK、连接、凭据与默认生成模型”。

#### 8.3.2 页面视觉链路

页面应表达以下真实关系：

```text
火山方舟 Provider → Ark SDK → 账号连接 → Seedream 5.0 Pro → 默认图片模型
```

页面不能将上述对象折叠成一张无层级的通用表单。

#### 8.3.3 Provider 卡片

火山方舟 Provider 卡片展示：

- Provider 名称：“火山方舟”
- Provider 标识：`volcengine-ark`
- 服务区域/API 地址摘要
- SDK 状态
- 已创建连接数量
- 可用图片模型数量
- 主操作：“创建连接”

#### 8.3.4 SDK 状态卡

展示字段：

- SDK 名称：“Volcengine Python SDK / Ark”
- 安装状态：“已安装”“未安装”“版本不兼容”“检测失败”
- 当前版本，例如 `5.0.36`
- 目标版本，例如 `5.0.36`
- 最近检测时间
- 操作：“重新检测”

状态规则：

- 已安装且版本匹配：绿色状态“可用”。
- 未安装：阻止真实连接测试和生成，提供修复说明。
- 版本不兼容：展示当前版本和要求版本。
- 检测失败：展示安全的归一化错误，不展示本地敏感路径。

本期不允许用户输入任意包名、镜像地址或版本号。

#### 8.3.5 新建图片连接

表单字段：

| 字段 | 必填 | 默认值 | 规则 |
| --- | --- | --- | --- |
| Provider | 是 | 火山方舟 | 本期只允许内置 Provider |
| 连接名称 | 是 | 空 | 1–60 字符，项目内可区分 |
| API 地址 | 是 | 官方 Ark 地址 | 普通用户只读；高级模式下遵循后端 allowlist |
| API Key | 新建时是 | 空 | 提交后写入 OS Keyring，不回显 |
| 启用连接 | 否 | 开启 | 关闭后不能分配和生成 |

按钮：

- “保存并测试”作为主按钮。
- “仅保存”作为次按钮，适用于暂时无法联网的环境。
- “取消”关闭表单且不保留 API Key。

防重复规则：

- 保存期间按钮进入 loading 并禁用重复提交。
- API Key 不进入前端持久化、草稿恢复、日志或埋点。

#### 8.3.6 图片连接卡片

展示：

- 连接名称
- Provider
- 启用状态
- “凭据已配置/未配置”
- 最近测试状态与时间
- 使用中的模型
- 是否为默认图片模型

操作：

- “编辑”
- “更新凭据”
- “测试连接”
- “设为默认”
- “停用/启用”
- “删除”

删除前必须说明正在使用该连接的默认分配和画布节点数量；存在依赖时遵循后端冲突规则，不允许静默解除。

#### 8.3.7 Seedream 模型卡片

展示：

- 显示名称：“Doubao Seedream 5.0 Pro”
- 模型 ID：`doubao-seedream-5-0-pro-260628`
- Provider：“火山方舟”
- 能力标签：“文生图”“智能改图”“多图参考”“点选修改”“框选修改”“1K”“2K”
- 参考图上限：“最多 10 张”
- 输出：“单次 1 张”
- 不支持项放入说明：“不支持 4K、原生 mask、流式进度和取消”
- 操作：“设为默认图片模型”

#### 8.3.8 默认图片模型

- 用户文案使用“默认图片模型”。
- 选择项显示“连接名称 / Doubao Seedream 5.0 Pro”。
- 只允许选择已启用、凭据已配置且模型匹配的连接。
- 未分配时，画布入口仍可见，但进入配置引导。

### 8.4 P1：图片生成节点端口与连线

#### 8.4.1 输入端口

- “提示词”：最多一条输入边。
- “主图”：最多一张，用于智能改图和局部修改。
- “参考图”：最多十张，保留有序 `referenceId`。

#### 8.4.2 输出端口

- “生成结果”：输出当前本地 asset。
- 未生成成功时输出端口存在但不可作为有效图片来源。

#### 8.4.3 连线反馈

- 开始拖线时高亮合法端口。
- 非法端口不高亮，并在松开时显示原因。
- 重复提示词、重复主图、超过十张参考图时阻止写入项目。
- 成功连接后显示来源节点名称或图片缩略图。
- 参考图区域显示容量，例如“3/10”。

### 8.5 P1：提示词与快捷提示词

- 用户可以在 Inspector 直接输入提示词。
- 快捷提示词可以点击插入当前选中的图片生成节点。
- 快捷提示词可以拖拽到节点或提示词端口。
- 文本节点连接后，Inspector 显示“来自节点：{节点名称}”。
- 本地编辑内容与连接提示词的合并规则必须可见，不允许静默覆盖。
- 保存结构化 `PromptDocument`，不保存 contentEditable HTML。

### 8.6 P1：结构化 `@` 引用

- 输入 `@` 打开当前节点可用图片选择器。
- 选择后生成可识别 token，例如 `@产品图`。
- token 绑定稳定 `referenceId`，显示名可随资产标题更新。
- 参考图排序后 token 仍绑定原资产，编译图号按当前顺序变化。
- Inspector 提供编译提示：“产品图将作为图1”。
- 参考图删除或断线后，失效 token 显示错误态并阻止生成。
- 不允许将 `@` 文本原样传给供应商。

### 8.7 P1：参考图管理

参考图列表展示：

- 缩略图
- 图号，例如“图1”
- 用户标签
- 来源资产或来源节点
- 被哪些 `@` token 引用
- 拖拽排序手柄
- “移除”操作

规则：

- 排序更新 `inputOrder`，不更换 `referenceId`。
- 主图和参考图角色必须可区分。
- 同一资产是否允许同时作为主图和参考图：待确认。

### 8.8 P1：生成模式

界面文案：

- “文生图”
- “参考图生成”
- “智能改图”
- “局部修改”

前置条件：

| 模式 | 必需输入 | 可选输入 |
| --- | --- | --- |
| 文生图 | 提示词 | 无 |
| 参考图生成 | 提示词、至少一张参考图 | 更多参考图 |
| 智能改图 | 提示词、主图 | 参考图 |
| 局部修改 | 提示词、主图、至少一个区域 | 参考图 |

不满足条件时禁用“生成”，并在按钮附近显示缺失项。

### 8.9 P1：尺寸与输出设置

- 比例：“智能”“1:1”“4:3”“3:4”“16:9”“9:16”“3:2”“2:3”“21:9”“自定义”。
- 分辨率：“1K”“2K”。
- 自定义尺寸：宽、高输入框和比例锁定。
- 输出格式：“PNG”“JPEG”。
- 水印：“开启/关闭”。
- 只展示当前模型声明支持的选项。
- 自定义尺寸错误就地显示，不在 API 返回后才提示。
- 当比例预设生效时，不持久化无效 width/height。

### 8.10 P1：区域修改

- 从节点或 Inspector 点击“编辑修改区域”打开大图编辑器。
- 工具：“点选”“框选”“选择/移动”“删除”“撤销”“重做”“适应画面”“缩放”。
- 每个区域绑定主图或指定参考图。
- 点选和框选坐标持久化为 0–999 整数。
- 框选必须满足 `x1 < x2`、`y1 < y2`。
- 提交后 Inspector 展示区域数量和摘要。
- 固定说明：“区域修改使用点/框指代，不是原生蒙版上传”。

### 8.11 P1：生成状态与错误恢复

#### 8.11.1 节点状态

- “待生成”
- “校验中”
- “生成中”
- “生成成功”
- “生成失败”

#### 8.11.2 状态规则

- 点击生成后立即禁用重复提交。
- 不显示虚假百分比。
- 不显示供应商不支持的取消按钮。
- 重试和再次生成始终创建新 run。
- 项目切换后，旧项目请求不得更新当前项目节点。

#### 8.11.3 错误与恢复动作

| 错误类型 | 用户提示 | 主操作 |
| --- | --- | --- |
| 未配置默认模型 | 尚未配置图片生成模型 | 前往配置 |
| 凭据缺失 | 连接尚未配置 API Key | 更新凭据 |
| 连接停用 | 当前图片连接已停用 | 前往连接 |
| 输入缺失 | 缺少主图/参考图/区域 | 返回节点补充 |
| 尺寸不合法 | 当前尺寸不受模型支持 | 修改尺寸 |
| 并发占满 | 当前生成任务较多 | 稍后重试 |
| 供应商限流/超时 | 供应商暂时不可用 | 重试 |
| 输出校验失败 | 返回图片未通过安全校验 | 查看详情/重试 |
| 存储失败 | 结果无法写入本地资产库 | 检查存储状态 |

前端不得显示真实密钥、Authorization、供应商临时 URL、URL query、本地绝对路径或原始异常堆栈。

### 8.12 P1：结果操作

生成成功后提供：

- “查看大图”
- “再次生成”
- “智能改图”
- “作为新图片节点放入画布”
- “作为参考图连接到新生成节点”
- “查看本次历史”
- “在媒体库中查看”

若系统已自动保存媒体库，界面显示“已保存到媒体库”，不再提供会造成重复资产的“保存”按钮。

### 8.13 P1：生成历史

- 支持“当前节点”和“当前项目”两个范围。
- 支持状态筛选：“全部”“成功”“失败”“生成中”。
- 历史项展示时间、模型、连接、模式、尺寸、提示词摘要、输入图和输出图。
- 失败项展示归一化错误和“重试”。
- 成功项支持“设为节点当前结果”“放入画布”“在媒体库查看”“再次生成”。
- 历史无普通删除入口。
- 项目或节点删除后，历史仍由全局历史入口查询；全局入口位置待确认。

### 8.14 P1：媒体库闭环

生成结果在媒体库展示：

- “AI 生成”标识
- 模型名称
- 生成时间
- 来源项目和节点
- 尺寸和格式
- “查看生成参数”

拖入画布时提供：

- “作为普通图片”
- “作为图片生成参考图”

选择“作为图片生成参考图”时：

- 若画布已选中生成节点，连接到该节点。
- 若没有选中生成节点，允许新建生成节点并连接。

### 8.15 P2：跨页面返回与上下文保持

- 从未配置节点跳转模型管理时记录来源项目和节点。
- 配置完成后提供“返回图片生成节点”。
- 返回后自动刷新模型目录、连接和默认图片模型。
- 若原节点已删除，返回当前项目并提示节点不存在。

### 8.16 Feature Flag 交互

- 前端用户设置：`imageGenerationNodeV1`。
- Runtime 环境：`PROMPTCARD_IMAGE_GENERATION_NODE_V1=true`。
- 开发/测试环境应提供明确启用方式并在诊断区域展示两道开关状态。
- 前端开启、Runtime 关闭时，入口显示但生成按钮禁用，提示“图片生成服务尚未启用”。
- Runtime 开启、前端关闭时，不展示创建入口，但已存在节点仍需安全加载并保持历史可读。
- 不允许把 Feature Flag 关闭表现为数据丢失或未知节点。

## 12. 非功能需求

### 12.1 性能

- 模型管理首屏在本地 Runtime 健康时应在 1 秒内呈现基础骨架和已缓存非敏感数据。
- 图片节点创建不触发 SDK、凭据或供应商请求。
- 历史列表使用分页，不一次加载全部永久历史。
- 大图、参考图和媒体库使用缩略图，按需加载原图。

### 12.2 安全

- API Key 输入使用密码字段，可临时显示，但不得复制到日志或持久化前端状态。
- 保存成功后清空输入值，只展示“已配置”。
- 页面刷新和返回不得恢复 API Key 明文。
- SDK 管理操作必须使用后端 allowlist，不接受任意包名或命令。
- 所有错误经过脱敏。

### 12.3 可用性

- 所有异步操作提供 loading、成功和失败反馈。
- 所有禁用按钮提供原因。
- 空状态必须包含下一步主操作。
- 页面跳转保留返回路径。

### 12.4 可访问性

- 工具栏图标必须有可访问名称和 Tooltip。
- 键盘可以创建节点、进入 Inspector、切换模式、排序参考图和执行生成。
- 焦点不能被画布快捷键抢占。
- 文字和背景对比度满足 WCAG AA；当前浅灰辅助文字需调整。
- 状态不能只依赖颜色，必须包含文字或图标。
- Dialog 打开后管理焦点，关闭后返回触发按钮。
- 拖拽排序提供键盘替代操作。

### 12.5 兼容性

- 首期桌面端 Windows 为主要验收环境。
- 浏览器开发模式与 Tauri WebView 行为一致。
- 旧项目中未知或关闭 Feature Flag 的图片节点不丢失。

## 15. 验收标准

### 15.1 画布入口

- Given 图片功能已开放，When 用户进入自由画布，Then 工具栏显示“图片生成”。
- Given 尚未配置默认图片模型，When 用户点击“图片生成”，Then 创建引导态节点并提供“前往配置”。
- Given 已配置默认图片模型，When 用户点击“图片生成”，Then 在视口中心创建有效节点、自动选中并打开 Inspector。
- Given Feature Flag 关闭但项目已有图片节点，When 用户加载项目，Then 节点数据不丢失且历史仍可读。

### 15.2 模型管理

- Given 用户进入 Agent 面板，Then 可以分别看到“文字模型”和“图片生成模型”。
- Given 用户进入图片生成模型页，Then 可以识别 Provider、SDK、连接、模型和默认分配五层关系。
- Given 用户提交 API Key，When 保存成功，Then 页面只显示“凭据已配置”，刷新后不回显密钥。
- Given SDK 未安装或版本不匹配，When 用户查看页面，Then 页面显示明确状态并阻止测试/生成。
- Given 火山连接测试成功，When 用户设为默认，Then “默认图片模型”显示连接名称和 Seedream 模型。
- Given 连接仍被使用，When 用户删除，Then 前端显示依赖和冲突，不静默解除。

### 15.3 节点输入与参数

- Given 节点已有一条提示词连接，When 再连接第二条，Then 连接被拒绝且项目不写入无效边。
- Given 节点已有十张参考图，When 连接第十一张，Then 连接被拒绝并显示上限。
- Given 用户调整参考图顺序，Then `@` token 仍指向原资产，图号按新顺序更新。
- Given token 对应参考图被删除，Then token 显示失效态且生成被阻止。
- Given 用户选择局部修改但未创建区域，Then 生成按钮禁用并说明缺少区域。
- Given 用户选择 2K 和合法比例，Then请求和历史快照保留该设置。

### 15.4 生成与错误

- Given 输入合法且连接可用，When 用户点击生成，Then 状态依次显示校验中、生成中和终态。
- Given 用户连续点击生成，Then 只创建一次请求。
- Given 凭据缺失，When 用户生成，Then 显示“更新凭据”操作且不泄露密钥信息。
- Given 供应商限流，When run 失败，Then 显示可重试提示且历史保留失败记录。
- Given 用户点击重试，Then 创建新 run，不覆盖失败 run。
- Given 用户切换项目，When旧项目请求完成，Then 不更新当前项目节点。

### 15.5 结果、历史与媒体

- Given 生成成功，Then 节点显示本地结果，历史新增成功 run，媒体库新增 AI 生成资产。
- Given 用户从历史选择旧结果，When 设为当前结果，Then 节点展示该资产且历史不被修改。
- Given 用户从媒体库拖入生成资产，Then 可选择作为普通图片或参考图。
- Given 项目或节点被删除，Then 永久历史仍可由保留入口查询。

### 15.6 可访问性与文案

- Given 用户仅使用键盘，Then 可以创建节点、配置模型、编辑参数并发起生成。
- Given 用户使用屏幕阅读器，Then 工具栏图标、状态和错误都有可识别名称。
- Given 页面语言为中文，Then 除技术标识外不出现 `Runtime ready`、`chat.primary` 等未本地化主文案。
- Given 辅助文字和禁用状态，Then 对比度满足 WCAG AA 或提供等价可读表达。

## 17. 依赖与风险

### 17.1 依赖

- Agent Runtime 模型目录、连接、assignment 和图片生成 API。
- OS Keyring 可用。
- `volcengine-python-sdk[ark]==5.0.36`。
- PromptCard Storage schema v3、asset、recent capture 和永久历史。
- 自由画布类型化节点和边。
- Feature Flag 配置和诊断能力。

### 17.2 风险

- 双 Feature Flag 导致“代码存在但用户看不到”再次发生。
- SDK 状态、连接测试和模型可用性混为一个状态，造成错误判断。
- 模型管理拆页后出现连接数据重复或默认分配不一致。
- 多参考图排序和 token 绑定在 UI 层发生错位。
- 项目切换时异步响应污染当前节点。
- API Key 被表单缓存、自动填充、错误采集或埋点意外记录。
- 生成历史永久保留带来磁盘容量增长，需要提供容量说明但不能增加普通删除入口。
- 真实 Ark 账号的权限、配额、区域和模型 ID可能导致集成环境与测试环境差异。

## 19. 设计与原型参考

### 19.1 参考方向

- 主参考：`figma`，用于画布工具栏、节点选择、右侧 Inspector 和拖拽连接的创作工具逻辑。
- 辅助参考：`replicate`，用于 Provider、模型能力、运行状态和开发者信任信息的组织。
- 对照参考：`linear.app`，用于高密度设置页面、状态反馈和低干扰操作；避免将其列表模式生硬复制到画布。

### 19.2 原创设计方向

- 视觉氛围：延续 PMAgent 当前浅色、低干扰工作台，不引入通用 AI 紫蓝渐变。
- 布局：画布保持最大可用面积；高频操作留在节点，低频配置进入 Inspector；模型管理采用窄导航、主列表/卡片、右侧详情或抽屉。
- 组件：使用状态标签、连接卡、能力标签、步骤关系、就地错误和空状态主操作，减少大面积无层级卡片。
- 色彩：中性背景为主；黄色延续品牌强调；绿色/红色/橙色只用于成功、错误、警告。
- 字体：统一高可读无衬线字体；正文不小于 14px；模型 ID和 SDK 版本可使用等宽字体。
- 动效：150–250ms，仅用于节点创建、Inspector 打开、状态变化和连接反馈。

### 19.3 设计旋钮

- 差异度：`6/10`
- 动效强度：`4/10`
- 信息密度：画布 `5/10`，模型管理 `7/10`

### 19.4 必须避免

- 隐藏核心入口来代替禁用和引导。
- 将所有模型类型放进一张通用表单。
- 只用颜色表示连接或 SDK 状态。
- 用内部槽位、ID 和后端错误码作为主要文案。
- 堆叠大量营销式卡片和装饰渐变。
- 伪造生成进度、取消、4K、mask 或多图能力。
- API Key 回显、浏览器持久化或埋点采集。

## 20. 发布门禁

- P0、P1 所有 Given/When/Then 验收通过。
- 真实 Windows Credential Locker 验证通过。
- Ark SDK 状态与版本从真实 Runtime 返回，不使用前端硬编码假状态。
- 使用 fake Provider 完成稳定 E2E。
- 使用真实 Seedream 账号完成文生图、多参考图、智能改图、point 和 bbox 五组冒烟。
- 关闭任意 Feature Flag 后，旧节点、历史和媒体资产仍可读。
- 重启桌面应用后连接、节点、历史和结果恢复。
- secret 扫描、日志扫描和错误脱敏检查通过。
- TypeScript、Storage、Agent Runtime、Rust、Playwright 和构建门禁全部通过。
